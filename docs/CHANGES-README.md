# دليل تطبيق هذي الحزمة

المطلوب: عند الضغط على محاضرة، بدل فتح عارض ملء شاشة (كان يوهم مستخدم الهاتف أن الملف = صفحة
واحدة فقط)، تنسدل قائمة صغيرة أسفل المحاضرة مباشرة فيها زران فقط: "تنزيل" و"فتح بتبويب جديد".
فتح محاضرة ثانية يغلق الأولى تلقائياً (أكورديون — عنصر واحد مفتوح بكل الصفحة).

## 1. انسخ فوق الملفين التاليين (استبدال كامل)

```
src/pages/Subject.jsx
src/components/subject/LectureItem.jsx
docs/logs/member-2-log.md
```

## 2. ⚠️ احذف هذا الملف (لم يعد مستخدَماً إطلاقاً بعد التعديل)

```bash
git rm src/components/subject/FileViewer.jsx
```

## 3. بعد ذلك، بجذر المستودع

```bash
git add -A
git commit -m "fix(ui): استبدال عارض ملء الشاشة بقائمة منسدلة (تنزيل/فتح بتبويب جديد) لحل توهّم مستخدم الهاتف"
git push
```

## ملاحظات

- السلوك الأكورديوني (فتح محاضرة يغلق أي محاضرة مفتوحة سابقاً) **لم يتغيّر** — كان موجوداً أصلاً
  بـ `Subject.jsx` (حالة `openFile` بمفتاح واحد فقط)، فقط بُسِّط لتخزين المفتاح النصي وحده
  (`openKey`) بدل كائن كامل، بما إن `LectureItem` صار يبني الروابط بنفسه.
- عناصر `link` و`note` (رابط خارجي / ملاحظة نصية) **لم تتأثر إطلاقاً** — ما كان لها علاقة بالعارض
  القديم أصلاً.
- زر التنزيل يستخدم `download` attribute (يفرض تنزيل الملف مباشرة بدل فتحه بالمتصفح)، وزر "فتح
  بتبويب جديد" يستخدم `target="_blank"` — نفس آلية الرابطين اللي كانا بأعلى العارض القديم بالضبط،
  فقط انتقلا لمكان أوضح (قائمة مباشرة تحت اسم المحاضرة، مو بأعلى شاشة كاملة قد تلتبس بمحتوى الملف).

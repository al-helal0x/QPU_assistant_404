## [التاريخ: 2026-07-20 — تحرير خطة المواد + حذف مباشر: نصيبي (دالة تحقق صغيرة)]

### ما تم إنجازه
- أضفت **`validateStudyPlan(courses)`** بملف `lib/idSlug.js` (المملوك لي أصلاً، لا ملف جديد):
  يتحقق من قائمة `study-plan.json` **كاملة** دفعة واحدة — أن `courses` مصفوفة فعلاً، صحة صيغة كل
  `id` (نفس نمط `isValidSlug`)، عدم تكرار أي `id` ضمن القائمة نفسها، ووجود `name` غير فارغ لكل
  عنصر. ترجع `{ valid: boolean, errors: string[] }`.
- هذي دالة **مكمِّلة** لا بديلة عن `isValidSlug` الموجودة أصلاً والمستخدَمة فعلياً بـ
  `StudyPlanEditor.jsx` (عضو 3) عند إضافة سطر واحد يدوياً — `isValidSlug` يفحص سطراً واحداً
  لحظة كتابته، و`validateStudyPlan` يفحص القائمة الكاملة كخط دفاع أخير قبل النشر (يلتقط مثلاً
  تكرار id ناتج عن مصدر غير الإضافة اليدوية بالواجهة، لو حدث مستقبلاً).
- اختبرتها: على `study-plan.json` الحقيقي الحالي (صالحة)، وعلى حالات مصطنعة (id مكرر، id بصيغة
  خاطئة، اسم مفقود، مدخل غير مصفوفة أصلاً) — كل الحالات أعطت رسائل خطأ واضحة ومحدَّدة بالعنصر.
- فحصت `node --check` على `idSlug.js` للتأكد من سلامة الصياغة.

### مشكلات واجهتها
- لا شيء يُذكر.

### قرارات اتخذتها (ولماذا)
- **لم أعدّل منطق `isValidSlug` نفسه** ولم أُضِف تحققاً إضافياً بخصوص "تعارض slug الحذف مع مادة
  نشطة" لأن التحليل انتهى إلى أن هذا مغطّى فعلياً بالتصميم الحالي: الـ id هو المفتاح الوحيد أصلاً،
  و`validateStudyPlan` الجديدة تضمن عدم وجود تكرار id بالقائمة من الأساس — بمجرد إزالة سطر (حذف من
  الخطة)، لا يمكن لسطر آخر أن "يتعارض" مع id لم يعد موجوداً بالقائمة إطلاقاً. لم أرَ حاجة فعلية
  لمنطق إضافي بلا سيناريو ملموس يبرره.
- اخترت وضع الدالة الجديدة بنفس `idSlug.js` بدل ملف منفصل، لأنها تتشارك نمط الـ id/slug نفسه مع
  الدوال الموجودة، ولتقليل عدد الملفات الجديدة بلا داعٍ فعلي.

### هل احتجت تعديل ملف خارج نطاق ملكيتك؟ (يُفترض دائماً "لا" بهذا التصميم)
- لا. التعديل بالكامل داخل `lib/idSlug.js` (ملفي).

### ملاحظات للمدير
- `validateStudyPlan` جاهزة للاستخدام الآن من عضو 3 (مثلاً كخطوة تحقق قبل بناء `pkg` بـ
  `StudyPlanEditor.jsx`، أو حتى من عضو 5 داخل `buildStudyPlanUpdate` نفسها كطبقة أمان إضافية قبل
  الكتابة الفعلية) — لا تحتاج تنسيقاً إضافياً معي، الاستيراد مباشر: `import { validateStudyPlan }
  from "../../lib/idSlug.js"`.
- لا عمل إضافي لي بخصوص `buildSubjectDeletion`/حذف الملفات على GitHub — هذا بالكامل نطاق عضو 5.

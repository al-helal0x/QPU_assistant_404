## 1. `subject.json`

مطابق تماماً لعقد §4.3 بخطة البناء، ومطابق فعلياً لبيانات البذرة (`public/data/subjects/*/subject.json`)
ولمخرجات `buildSubjectPackage()` (عضو 5):

```json
{
  "id": "circuits",
  "name": "الدارات الكهربائية",
  "code": "1130700",
  "hidden": false,
  "professorVariants": [
    { "professorId": "prof-ahmad", "professorName": "د. أحمد", "active": true, "lecturesFile": "lectures-prof-ahmad.json" }
  ]
}
```
- `professorVariants` اختياري تماماً — غيابه (كما في `subjects/database/subject.json`) يعني القراءة من `lectures.json` مباشرة.
- نشِط واحد فقط (`active: true`) بأي لحظة ضمن `professorVariants` — يضمنه `buildSubjectPackage()` تلقائياً.

## 2. `lectures.json` / `lectures-{professorId}.json`

```json
{
  "sections": [
    {
      "section": "theory",
      "hidden": false,
      "items": [
        { "title": "المحاضرة 1 — مقدمة", "file": "lecture-01.pdf", "hidden": false }
      ]
    }
  ]
}
```
- `section` أحد مفاتيح `SECTION_LABELS` (§4.4): `theory | lab | extra | exam`.
- `hidden` على 3 مستويات فعلياً بهذا الشكل: مادة (`subject.json`) → قسم (كائن القسم) → عنصر (كل محاضرة).
- تسمية ملفات PDF: `lecture-NN.pdf` برقم تسلسلي عام يكمل من أعلى رقم موجود بأي قسم (قرار عضو 5، موثّق بسجله).

## 3. `study-plan.json`

{

## 4. عقد `buildSubjectPackage({ subjectMeta, files }) → pkg` (عضو 5، `lib/githubPublisher.js`)

راجعت الكود مقابل التعليق التوثيقي أعلى الملف — مطابقان. ملخص:

- **الدخل** `subjectMeta`: `{ id, name, code, hidden?, professorId?, professorName?, setActive?, existingSubject?, existingLectures? }`
- **الدخل** `files`: `Array<{ file: File, title, section: "theory"|"lab"|"extra"|"exam", hidden? }>`
- **الخرج** `pkg`: `{ slug, subjectPath, lecturesPath, pdfDir, subjectJson, lecturesJson, pdfFiles: [{ path, file, name }] }`
- هذا هو العقد الذي يجب أن يبنيه عضو 3 ضده عند استبدال Placeholder الخاص بـ `SubjectForm.jsx` — يستدعي `buildSubjectPackage()` بنفسه ثم يمرر الناتج كما هو لـ `<PublishPanel pkg={pkg} />` (لا يحتاج معرفة أي تفصيل عن GitHub API).

## 5. `publishToGitHub({ token, owner, repo, pkg, baseBranch? }) → { prUrl, branch }`

- يفتح فرع `content/{slug}-{timestamp}` ثم PR إلى `main` (بدون دمج تلقائي — طبقة أمان مقصودة).
- التوكن يُستخدم من الذاكرة فقط (`useState` بمكوّن React)، لا يُخزَّن ولا يُرسَل لغير `api.github.com`. ✅ تحقّقت من الكود، لا يوجد أي `localStorage`/`sessionStorage` أو استدعاء لغير GitHub API.

## 6. `exportPackageAsZip(pkg) → Blob`

- يعتمد `jszip` (انظر ملاحظة القسم 7 أدناه).

## 1. `subject.json`

مطابق تماماً لعقد §4.3 بخطة البناء، ومطابق فعلياً لبيانات البذرة (`public/data/subjects/*/subject.json`)
ولمخرجات `buildSubjectPackage()` (عضو 5):

```json
{
  "id": "circuits",
  "name": "الدارات الكهربائية",
  "code": "1130700",
  "hidden": false,
  "professorVariants": [
    { "professorId": "prof-ahmad", "professorName": "د. أحمد", "active": true, "lecturesFile": "lectures-prof-ahmad.json" }
  ]
}
```
- `professorVariants` اختياري تماماً — غيابه (كما في `subjects/database/subject.json`) يعني القراءة من `lectures.json` مباشرة.
- نشِط واحد فقط (`active: true`) بأي لحظة ضمن `professorVariants` — يضمنه `buildSubjectPackage()` تلقائياً.

## 2. `lectures.json` / `lectures-{professorId}.json`

```json
{
  "sections": [
    {
      "section": "theory",
      "hidden": false,
      "items": [
        { "title": "المحاضرة 1 — مقدمة", "file": "lecture-01.pdf", "hidden": false }
      ]
    }
  ]
}
```
- `section` أحد مفاتيح `SECTION_LABELS` (§4.4): `theory | lab | extra | exam`.
- `hidden` على 3 مستويات فعلياً بهذا الشكل: مادة (`subject.json`) → قسم (كائن القسم) → عنصر (كل محاضرة).
- تسمية ملفات PDF: `lecture-NN.pdf` برقم تسلسلي عام يكمل من أعلى رقم موجود بأي قسم (قرار عضو 5، موثّق بسجله).

## 4. عقد `buildSubjectPackage({ subjectMeta, items }) → pkg` (عضو 5، `lib/githubPublisher.js`)

> ⚠️ **مُحدَّث 2026-07-21:** المعامل تغيّر من `files` إلى `items` (يدعم الآن `pdf`/`image`/`link`/`note`،
> لا الملفات فقط). التفاصيل الكاملة بالقسم 13 أدناه. الملخص هنا مُحدَّث ليطابق الوضع الحالي فعلياً.

راجعت الكود مقابل التعليق التوثيقي أعلى الملف — مطابقان. ملخص:

- **الدخل** `subjectMeta`: `{ id, name, code, hidden?, professorId?, professorName?, setActive?, existingSubject?, existingLectures?, existingStudyPlan? }`
- **الدخل** `items`: `Array<{ type?, title, section: "theory"|"lab"|"extra"|"exam", hidden?, file?, url?, content? }>` — انظر القسم 13 للشكل الكامل حسب `type`.
- **الخرج** `pkg`: `{ slug, subjectPath, lecturesPath, pdfDir, studyPlanPath, studyPlanJson, subjectJson, lecturesJson, pdfFiles: [{ path, file, name }] }` — `pdfFiles` الآن يشمل عناصر `image` أيضاً، لا `pdf` فقط.
- هذا هو العقد الذي يبني عليه عضو 3 بـ `SubjectForm.jsx` — يستدعي `buildSubjectPackage()` بنفسه ثم يمرر الناتج كما هو لـ `<PublishPanel pkg={pkg} />` (لا يحتاج معرفة أي تفصيل عن GitHub API).

## 13. مخطط عنصر `items` — أنواع متعددة (pdf/image/link/note)

> مصدر هذا القسم: العقد النصي المتفق عليه بين عضو 3 وعضو 5 (2026-07-21)، حوّله عضو 4 لتوثيق
> منظّم، وأدمجه هنا الآن بعد التحقق من مطابقة تسليمات عضو 2، عضو 3، عضو 5 له فعلياً (القسم 14).

### 13.1 شكل العنصر داخل مصفوفة `items`

يُمرَّر إلى `buildSubjectPackage({ subjectMeta, items })`.

| الحقل | النوع | إلزامي؟ | يُستخدم مع | ملاحظات |
|---|---|---|---|---|
| `type` | `"pdf"` \| `"image"` \| `"link"` \| `"note"` | اختياري | الكل | **غيابه = `"pdf"` دائماً**، بلا استثناء (توافق عكسي مع البيانات القديمة) |
| `title` | `string` | إلزامي | الكل | |
| `section` | `"theory"` \| `"lab"` \| `"extra"` \| `"exam"` | إلزامي | الكل | نفس القيم الحالية، بلا أي تغيير |
| `hidden` | `boolean` | إلزامي | الكل | |
| `file` | `File` | إلزامي | `pdf`, `image` فقط | كائن `File` فعلي قادم من المتصفح |
| `url` | `string` | إلزامي | `link` فقط | يجب أن يبدأ بـ `http://` أو `https://` |
| `content` | `string` | إلزامي | `note` فقط | نص خام فقط — **ممنوع Markdown أو HTML** |

### 13.2 قواعد ثابتة (تُطبَّق من الطرفين معاً، لا طرف واحد)

1. **التحقق من `url`** يُطبَّق مزدوجاً: بواجهة عضو 3 (`SubjectForm.jsx`، دالة `isValidUrl`) وبكود عضو 5 (`buildSubjectPackage`، دالة `isValidHttpUrl`) — كلاهما بنفس الـ regex فعلياً (`/^https?:\/\//i`)، تحقّقت منه.
2. **صيغ `image` المقبولة:** `png` / `jpeg` / `webp` فقط. **لا SVG.** مُطبَّقة بالطرفين: فلترة `accept`/drag-drop بـ `FileUploaderWidget.jsx` (عضو 3)، ورمي خطأ صريح بـ `buildSubjectPackage` (عضو 5) لو مرّت صورة SVG رغم ذلك.
3. **مسار الملفات الفعلية** يبقى `public/pdf/{slug}/` بلا تغيير اسم المجلد — حتى لعناصر `image`.
4. **عنصر بدون `type`** بالبيانات القديمة يُعامَل كـ `"pdf"` دائماً — مُطبَّق بالثلاثة أطراف: `buildSubjectPackage` (البناء)، و`LectureItem.jsx`/`FileViewer.jsx`/`Subject.jsx` (العرض، عضو 2).

### 13.3 سلوك كل type عند البناء (`buildSubjectPackage`)

| type | سلوك الرفع |
|---|---|
| `pdf` / `image` | نفس منطق رفع الملف الحالي (اشتقاق اسم الملف من `title` + معالجة تصادم الأسماء)، مع فرض امتداد يطابق النوع فعلياً |
| `link` / `note` | **لا رفع ملف إطلاقاً** — الحقل يُكتب مباشرة بعنصر `lectures.json` |

### 13.4 سلوك كل type عند العرض (`LectureItem.jsx` / `FileViewer.jsx` / `Subject.jsx`)

| type | طريقة العرض |
|---|---|
| `pdf` | كما هو حالياً (بلا تغيير) — عارض ملء شاشة عبر `FileViewer` |
| `image` | عارض ملء شاشة مشابه (نفس `FileViewer`، فرع `<img>` بدل `<iframe>`) |
| `link` | زر مباشر بلا عارض منفصل، `target="_blank"` — لا علاقة له بحالة "أي عنصر مفتوح" بالأب |
| `note` | توگل محلي داخل `LectureItem` نفسه (لا يعرفه الأب)، يعرض `content` inline (`white-space: pre-wrap`)، بلا عارض منفصل |

### 13.5 أمثلة

```js
// pdf
{ type: "pdf", title: "محاضرة 1", section: "theory", hidden: false, file: File }
{ title: "محاضرة 1", section: "theory", hidden: false, file: File } // type غائب = pdf

// image
{ type: "image", title: "مخطط الدارة", section: "lab", hidden: false, file: File }

// link
{ type: "link", title: "فيديو شرح خارجي", section: "extra", hidden: false, url: "https://example.com/video" }

// note
{ type: "note", title: "ملاحظة سريعة", section: "exam", hidden: false, content: "راجع الفصل 3 فقط، بلا الفصل 4." }
```

### 13.6 شكل `lectures.json` بعد هذا التحديث (مثال مختلط)

```json
{
  "sections": [
    { "section": "theory", "hidden": false, "items": [
      { "type": "pdf", "title": "محاضرة 1", "file": "محاضرة_1.pdf", "hidden": false }
    ]},
    { "section": "lab", "hidden": false, "items": [
      { "type": "image", "title": "مخطط الدارة", "file": "مخطط_الدارة.png", "hidden": false }
    ]},
    { "section": "extra", "hidden": false, "items": [
      { "type": "link", "title": "فيديو خارجي", "url": "https://example.com/v", "hidden": false }
    ]},
    { "section": "exam", "hidden": false, "items": [
      { "type": "note", "title": "ملاحظة", "content": "راجع الفصل 3", "hidden": false }
    ]}
  ]
}
```
عناصر `pdf`/`image` تحمل `file` (اسم فعلي بمجلد `public/pdf/{slug}/`)، `link` يحمل `url`، `note` يحمل `content` — بلا حقل `file` لكليهما.

## 17. 🔒 قاعدة إلزامية: ASCII فقط لـ `professorId` وأسماء الملفات الفعلية — 2026-07-22

عقب الخلل الحرج الموثَّق بالقسم 15 (فقدان ربط صامت بين `subject.json` والملف الفعلي على القرص
بسبب تلف ترميز أحرف عربية بأسماء الملفات)، اعتُمدت هذه القاعدة رسمياً بديلاً كاملاً لأي نص سابق
يخص `professorId` بالقسمين 3/13:

- **`id` المادة:** slug إنجليزي وصفي دائماً (كما كان — `circuits`, `database`).
- **`professorId`:** نفس القاعدة تماماً — **ASCII slug إلزامي دائماً بلا استثناء** (مثال:
  `prof-alaa`، لا `"د."` ولا `"الدكتور-علاء"`). `professorName` وحده يبقى بالعربي كنص عرض،
  ولا علاقة له بأي اسم ملف أو مسار.
- **أي ملف فعلي يُكتَب على القرص/الريبو** (PDF، صورة، `lectures-{professorId}.json`) يجب أن يكون
  ASCII بالكامل، بصرف النظر عن لغة `title` المعروض بالواجهة. الاسم العربي يبقى بحقل `title` بـ
  JSON فقط — لا يُستخدم أبداً حرفياً كاسم ملف.
- الرقم الرسمي (`code`) يبقى بحقل منفصل — لا يُستخدم بالروابط أو أسماء الملفات (كما كان).
- توليد الـ slug (لكل من `id` المادة و`professorId`) مسؤولية موحّدة بدالة عامة واحدة
  (`transliterateToSlug` بـ `lib/idSlug.js`، عضو 4) — قابل للتعديل اليدوي قبل الحفظ، والتحقق
  (`isValidSlug`) نفس الدالة بالضبط لكلا الحقلين، لا نسخة موازية.

### طبقات الإنفاذ الفعلية (الأربع، من الإدخال إلى النشر)

1. **الواجهة (`SubjectForm.jsx`، عضو 3):** `professorId` لدكتور جديد يُقترَح تلقائياً من
   `professorName` عبر `suggestProfessorId`، قابل للتعديل، ويُرفَض فوراً بالواجهة (رسالة تحت
   الحقل) لو غير ASCII أو مكرر ضمن نفس المادة — قبل حتى محاولة بناء حزمة النشر.
2. **بناء اسم ملف المحتوى (`sanitizeFileTitle`، عضو 5):** عنوان أي عنصر pdf/image يُحوَّل عبر
   `transliterateToSlug` قبل استخدامه كاسم ملف فعلي — العنوان العربي المعروض بالواجهة غير متأثر.
3. **بناء `lecturesFile` (`buildSubjectPackage`، عضو 5):** خط دفاع أخير صريح — يرفض النشر برسالة
   خطأ واضحة لو وصل `professorId` بأي حرف غير ASCII رغم الطبقتين أعلاه (مثلاً استدعاء مستقبلي
   للدالة من مسار غير الواجهة، كاختبار أو سكربت).
4. **تصدير Zip (`exportPackageAsZip`، عضو 5):** `platform: "UNIX"` صريح بخيارات JSZip — طبقة
   حماية إضافية تفرض ترميز UTF-8 قياسي بحقول ZIP metadata، حتى لو تسرَّب نص غير متوقَّع مستقبلاً.

### التحقق المستقل (عضو 6) بعد التنفيذ

- سيناريوهات Node مباشرة (بمعزل عن الواجهة) ضد `buildSubjectPackage`: `professorId` خام عربي
  يُرفَض برسالة واضحة؛ `professorId` مولَّد عبر `suggestProfessorId` ينتج مسارات ASCII بالكامل
  (تحقّقت بـ regex `^[\x00-\x7F]+$` على `lecturesPath` وكل مسار بـ `pdfFiles`)؛ تصادم اسمين
  متطابقين ينتج معرّفين فريدين بلاحقة رقمية، لا كتابة صامتة فوق دكتور موجود.
- بيانات البذرة الحية (12 مادة كانت متأثرة) أُصلِحت يدوياً بجولة سابقة مستقلة عن هذا التعديل
  الكودي — راجع القسم 15/16 للتفاصيل. هذا القسم يوثّق فقط منع تكرار نفس الخلل مستقبلاً عبر لوحة
  الإدارة الحية.
